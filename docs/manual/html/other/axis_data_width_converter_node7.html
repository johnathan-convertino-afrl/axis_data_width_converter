Architecture

The only module is the axis_data_width_converter module. It is listed below.

  • axis_data_width_converter Implement an algorithm to convert BUS data interfaces in even multiples (see core for documentation).

The data width converter uses a generate block to select between two possible scenarios. First is they are equal, which just assigns the fields to each other. The second is any difference in size. This will convert from one to another using a register build up method between the slave and master. If FLUSH is set and the sizes are different the core will flush out any bytes in the register when TLAST is valid.

  1. Create registers used to buffer data and signals
    1. READY Take input data and output data while increasing or decreasing register count.
    2. FULL Register is full of data and needs to have data removed before going back to ready.
    3. FLUSH If the FLUSH_LAST option is set, when TLAST is found in the ready state all valid tkeep data will be flushed from the core.
    4. OVERFLOW This indicates a bug in the core, clear all registers and increment overflow counter by 1 for each incident.
